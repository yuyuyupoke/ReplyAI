# YouTube API Quota Break Strategy (Lean Roadmap)

## 🎯 目的 (Goal)
**「月利30万円」**および**「EXIT」**を目指すにあたり、まずは**「最小限のリソースで最大の検証」**を行う（Lean Startup）。
複雑なBYOK（ユーザー自身のキー利用）システムは作らず、Google公式の申請プロセスを通じてスケールさせる。

---

## 🛣️ 戦略フェーズ (Strategy Phases)

### Phase 1: 初期検証 (The First Customer)
デフォルトのQuota (10,000 units/day) の範囲内で、**たった1人の熱狂的なテスト顧客**を満足させる。

*   **ターゲット**: 登録者1万〜5万人規模のYouTuber 1名。
*   **アクション**:
    *   現在のシステム（1日約190返信可能）で十分運用可能。
    *   まずはこの1名に徹底的に寄り添い、PMF（Product Market Fit）を確認する。
    *   **コスト**: 0円 (Default Quota)。

### Phase 2: 拡張申請 (Standard Extension)
テスト顧客が増え、デフォルトの10,000 unitsで足りなくなった段階で実行。

*   **トリガー**: 1日のAPI使用率が80%を超えた日が数日続いた時。
*   **アクション**:
    *   Google Cloud ConsoleからQuota Extension Requestフォームを提出する。
    *   一般的なWeb記事（"YouTube API quota extension request" 等で検索）を参考に申請。
    *   **目標**: 50,000 〜 100,000 units/day への引き上げ。

### Phase 3: クォータブレイク (Viral Scaling)
ツールがバズり、急激なユーザー流入が起きた段階（またはその予兆がある段階）で実行。

*   **トリガー**: Phase 2の枠でも足りないほどの急成長、または著名インフルエンサーによる紹介。
*   **アクション**:
    *   Googleの担当者へのコンタクトや、より詳細なAudit（監査）プロセスを経て、大規模な枠を確保する。
    *   この段階で初めて、法人化や資金調達、あるいはEXIT（事業売却）の交渉材料として「大規模なAPI枠」を活用する。

---

## 🚀 直近のアクションプラン (Next Steps)

1.  **テスト顧客の獲得**: Phase 1を実行するための「最初の1人」を見つける。
2.  **モニタリング**: 現在のAPI使用量を把握できるようにしておく（簡易的なログ確認でOK）。
3.  **機能開発**: その1人が快適に使えるための機能（マルチアカウント、通知など）に集中する。
